## 概要

**目的**: 投資家提案用デモ

**デザイン方針**: シンプル・モダン、データの可視化を重視

## スタイリング方針

- **CSSフレームワーク**: Tailwind CSS
- **カスタムスタイル**: `styles/styles.scss`（ネスト記法で一元管理）
- **カラー・フォント・間隔**: Tailwindのデフォルト設定を基本とし、必要に応じて`tailwind.config.js`でカスタマイズ

### ユーザータイプカラー（Tailwind クラス）
- イノベーター: `blue-500` / `bg-blue-500`
- コミッター: `emerald-500` / `bg-emerald-500`
- インベスター: `purple-500` / `bg-purple-500`

## 投資家向け訴求ポイント（画面別）

| 画面 | 訴求ポイント |
|------|-------------|
| LP | 課題認識と解決策の明確さ |
| PJ-LIST | マーケットの広がり、プロジェクトの多様性 |
| PJ-DETAIL | **差別化の核心**：実行体制可視化、Bounty Pool |
| PJ-CREATE | イノベーターのUX、参入障壁の低さ |
| CM-MATCH | 人材供給側の魅力、Boarding Trackの仕組み |
| DASHBOARD | 運用イメージ、進捗管理の透明性 |
