# 03. プロジェクト詳細 (PJ-DETAIL)

## 基本情報

| 項目 | 内容 |
|------|------|
| 画面ID | PJ-DETAIL |
| URL | `/projects/:id` |
| 主要ユーザー | 全員 |
| 目的 | プロジェクト理解、支援/参画意思決定 |

---

## 投資家への訴求ポイント（最重要画面）

- **差別化の核心**：実行体制の可視化
- **Bounty Pool**：資金使途の透明性
- **コミッター参画状況**：誰が何をするかが見える
- **クリティカルパス**：実行計画の具体性

## タブ詳細

### 1. 概要タブ

| セクション | 内容 |
|-----------|------|
| このプロジェクトについて | プロジェクトの詳細説明（リッチテキスト） |
| 解決したい課題 | 課題の箇条書き |
| 私たちの解決策 | 解決策の箇条書き |

### 2. 実行体制タブ（★最重要）

#### プロジェクトオーナー

| 項目 | 説明 |
|------|------|
| アバター | プロフィール画像 |
| 名前 | イノベーター名 |
| 所属 | 会社名/肩書き |
| 一言 | 自己紹介文（1行） |

#### 実行チーム（コミッター一覧）

| 状態 | 表示 |
|------|------|
| 確保済み（●） | コミッター情報、役割、Boardingレベル、一言 |
| 募集中（○） | 役割名、募集要件、[応募ボタン] |

#### クリティカルパス（タイムライン）

| 項目 | 表示内容 |
|------|---------|
| フェーズ名 | Phase番号 + 名称 |
| ステータス | 完了/進行中/予定 |
| 期間 | 予定年月 |
| 成果物 | 各フェーズの成果物 |

### 3. 資金使途タブ（★重要）

#### 使途内訳

| 項目 | 表示 |
|------|------|
| 円グラフ | カテゴリ別割合 |
| 製造費 | XX% |
| 実務枠（Bounty Pool） | XX% ← **強調表示** |
| 物流費 | XX% |
| その他 | XX% |

#### Bounty Pool詳細

| 表示項目 | 説明 |
|---------|------|
| 総額 | Bounty Poolの総額 |
| 役割別内訳 | 各役割への配分と状態（確定/募集中） |
| 支払いルール | 「CF成立後、成果物の検収に応じて支払い」 |

### 4. リターンタブ

リターンカード形式で表示。

| 項目 | 表示 |
|------|------|
| 金額 | ¥X,XXX |
| タイトル | リターン名 |
| 説明 | リターン内容 |
| 残数 | 残り XX/XX |
| お届け予定 | YYYY年MM月 |
| 支援ボタン | [支援する] |

### 5. 活動報告タブ

| 項目 | 内容 |
|------|------|
| 日付 | 投稿日 |
| タイトル | 報告タイトル |
| 本文 | 報告内容（プレビュー） |

## インタラクション

| 要素 | アクション | 遷移先/結果 |
|------|-----------|------------|
| 戻るリンク | クリック | `/projects` |
| タブ | クリック | タブ切り替え |
| 支援するボタン | クリック | 支援モーダル表示（モックではアラート） |
| スキルで参画するボタン | クリック | `/match` または 応募モーダル |
| この役割に応募するボタン | クリック | 応募モーダル表示 |
| リターンの支援するボタン | クリック | 支援モーダル（該当リターン選択済み） |
