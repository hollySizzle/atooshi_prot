## セクション詳細

### 1. Hero Section

| 要素 | 内容 |
|------|------|
| キャッチコピー | 「アイデアを、実行力でカタチにする」 |
| サブコピー | クラウドファンディング × 実務マッチングプラットフォーム |
| CTA（主） | プロジェクトを見る → `/projects` |
| CTA（副） | 始め方を知る → ページ内スクロール |
| 背景 | グラデーション or 抽象的なイラスト |

### 2. 課題提起セクション

3つの課題カード：

| # | タイトル | 説明 |
|---|----------|------|
| 1 | 資金だけでは成功しない | CFで資金を集めても、実行（製造・納品・CS）で躓くケースが多い |
| 2 | 実行力が見えない | 支援者はプロジェクトの実行体制を評価できない |
| 3 | 善意の搾取構造 | 「手伝います」が無償労働になりがち |

### 3. 解決策セクション

- 中央にエコシステム図（3者の関係図）
- アニメーションで流れを表現（モックでは静的でOK）

### 4. 3つのユーザータイプ

| ユーザータイプ | カラー | 説明 | CTA |
|---------------|--------|------|-----|
| イノベーター | `blue-500` | プロジェクトを立ち上げ、実行チームを組成 | プロジェクトを始める |
| コミッター | `emerald-500` | 専門スキルを活かして参画、継続的に関わる | スキルを活かす |
| インベスター | `purple-500` | 実行体制が見えるプロジェクトを支援 | プロジェクトを支援 |

### 5. 特徴・差別化セクション

| 特徴 | 説明 |
|------|------|
| Bounty Pool | CF調達額から実務枠を先に確保。未払いリスクを排除 |
| Boarding Track | 単発タスク→継続参画→ボーディングメンバーへの昇格パス |
| 実行体制可視化 | プロジェクトの「誰が何をするか」が支援前に分かる |

### 6. 注目プロジェクトセクション

- プロジェクトカード3枚（横並び）
- カード仕様は `02_プロジェクト一覧.md` 参照

### 7. 数字セクション

| 指標 | モックデータ |
|------|-------------|
| 累計プロジェクト | 123件 |
| 累計調達額 | ¥45,000,000 |
| 登録コミッター | 456名 |
